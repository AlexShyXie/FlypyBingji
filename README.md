## Rime并击、小鹤音形并击、最简单的并击

听闻并击很厉害，但是不知道为什么厉害，怎么厉害，于是在各个输入法群里咨询并在网上翻阅大量资料，我终于弄懂了并击是怎么回事。但凡搜索过怎么加快打字速度，都能听说这个并击方案，但是并击实在太小众了，资料极少，所以在这里做一个简单的学习总结，希望像我一样想了解的人，能直接看到我这一篇总结，少一点一脸茫然的时间。

个人猜测：**并击的起源**可以追溯到速录机专用方案。狭义的并击就是指速录机及其配套的打字方法、词库、编码方式和上屏方式。随着技术发展，出现了**26键盘可用的并击**方案，主要通过两种方式实现：

- 使用AutoHotkey软件（如[snochorded](https://github.com/snomiao/snochorded/)）

- 使用Rime输入法（如[rime-shuangpin-fuzhuma](https://github.com/gaboolic/rime-shuangpin-fuzhuma)）

本文重点讨论后者，因为这是大多数普通用户最容易接触到的并击方案。

### 并击的基本原理

**并击的核心原理**是将键盘左右两边都映射出26个字母，实现左手打声母、右手打韵母的分工。这种设计要求**手指具备良好的协调能力**，需要在11、12、21、22等指头模式下灵活操作。

对于**普通双拼改并击**的情况，**重码处理和上屏方式与常规双拼没有区别，仍然依赖原有的辅助码和候选选字机制**。这样的实现方式在速度提升上会有所限制，但是普通双拼改并击也许反而是大家想了解的。

### 速度与复杂度的权衡

如果追求极致速度，像著名的空明码这样的专业并击方案，就不是简单的按键映射了。它们**为了配套的重码方案和上屏方案，会对全部按键进行重新映射**。这也是为什么很多人搜索并击时，看到了空明码，然后进去看教程，就被复杂的教程劝退了。

作为小鹤音形用户，我发现打字速度快时经常出现按键顺序错误，因此开始研究并击。转向空明码成本太高，所以我继续探索普通双拼的并击方案。

### Rime中的并击实现

Rime提供了`chord_composer`模块直接支持并击：

```yaml
# 六、chord_composer_
# 并击把键盘分两半，相当于两块键盘
alphabet: # 字母表，击键按字母表顺序排列
algebra: # 拼写运算规则，转换为拼音音节
output_format: # 并击完成后套用的式样
prompt_format: # 并击过程中套用的式样
```

根据rime-shuangpin-fuzhuma方案，双拼并击可以实现：

- 串击与并击并存

- 正常打字模式

- 单手操作支持

这意味着**形码、辅助码、候选上屏、4码上屏**都可以保留，只需在现有方案基础上添加左右手映射即可。

### 小鹤音形并击实践

在小鹤音形方案中添加并击模块：

```yaml
engine:
  processors:
    - ascii_composer
    - chord_composer # 并击
    - ......
 
chord_composer: # 并击模块设置
  alphabet: "aqzswxdecfrvgtbpolikmujnyh'/"
  algebra:
    # 定义左手11键       
    - xform/^qw(?![左右])/p左/
    # ... 其他映射规则
```

**按键映射设计**遵循我个人舒适度原则。例如：

- 第一排尽量从两边往中间，每2个按键映射对面的一个键，qw映射p，po映射q

- 但是R和U的时候，感觉食指和中指一起按也能当作一个映射，所以就把ef映射成u，ji映射成r

- 第三排的m、n放在vx、vc上

- 部分按键做了些冗余。

**关键细节**：在方案里修改映射时，使用ji还是ij的写法，需要看alphabet的排列顺序。字母顺序设计为左手字母在前，确保并击出来的字符声母在前。

**实际操作示例**：

- 地(di)：左手声母，并击d和i

- 皮(pi)：p在右手需映射到左手(qw)，并击qwi

- 么(me)：m在右手，e在左手，并击vc出m，iu出e，vciu出么

上屏方式仍然是小鹤音形的4码上屏，没有任何区别。

### 实际使用中的挑战

这种方案容易理解，确实能加快打字速度，但是这样优势其实也有限，空明码并击的优点应该也是要和他们的重码方案配合后才能显现。

其主要挑战在于改变长期形成的打字习惯，串击和并击的区别：

比如：

所有人都习惯**串击**输入，比如打`ve 着`时，会无意识地先按v，然后在**不松开v的情况下按e**。这时系统会识别成并击，编码显示为ev（因为alphabet里e在v前面），这种习惯改变需要时间适应。

另外，虽然并击确实会避免很多难打的键位，但是单键打快了也会误识别为并击，单键的字也不能按太快，按空格上屏也不能太快。感觉如果能适应比较别扭的手法的话也许并击并不能提速。

> 这个问题已经解决，在alphabet里把 `,.;'空格` 都纳入就可以了，这样，正常的按键，再根据候选按下相应的上屏键，空格首选、 ;次选、'3选，
注意需要键盘4键以上无冲。

## 以下是针对并击优点的详细拆解：

### 1. 手部健康与舒适度（保护手指，降低疲劳）

这是大家最关注的优点，特别是对于高强度的文字工作者。

*   **核心原理：速度变慢 = 保护**

    *   **疑问**：并击时一次按下的键数可能变多了（比如双拼按2个键，并击可能同时按2-6个键），这不是更累吗？

    *   **解释**：其实你按键的**动作速度变慢了**。原来需要手指飞速运动才能打出的字，现在用较慢的手部动作就能完成。这种慢速动作对手指的冲击更小。

    *   **原话**：“你原来需要很快的速度才能打出来两个字，现在很慢的速度就能打出来两个字，这就是对手指的一种保护呀。”
    
*   **击键频率降低，休息时间增加**

    *   **解释**：虽然单次动作可能涉及多个手指，但单位时间内手部整体的运动**频率（Hz）降下来了**。频率一下降，手指和肌肉就有更多时间进行微休息。
    
    *   **原话**：“其实主要是频率下来了，频率下来，对手指休息就多了，这样比高频更保护手指。”
    
*   **发力方式不同（手掌下压次数减少）**

    *   **解释**：打字不仅是手指动，手掌也在发力。并击是双手同时协作，虽然平均每个字涉及的按键数增加，但手掌整体“下压”的总次数减少了。
    
    *   **细节**：拇指和食指同时按下的力，与先后分别按下的力消耗是不同的。并击减少了高频的反复机械运动。
    
    *   **原话1**：“虽然平均每个字的按键数增加了，但是手掌下压的次数少了。”
    
    *   **原话2**：“并击击键少，也对手很友好，击键高了很累手的，也很伤手。”
    
### 2. 效率与码长（速度快，性价比高）

*   **实际码长并不长**

    *   **解释**：很多人误以为并击码长很长，但实际上通过优化，字均按键量被控制得很好。
    
    *   **数据支持**：拼读并击的字均按键量约为2.4-2.5左右，这与传统的串击（如双拼）码长相差无几，但因为并发输入，速度却快得多。
    
    *   **原话**：“比如今天的赛文，拼读并击的理论字均按键量就是2.4。这也不比串击的码长高多少。”
    
*   **“一键字词”的优势**

    *   **解释**：并击方案中存在大量“一键出字”或“一键出词”的高频字，这极大地拉低了整体的平均按键量。
    
    *   **原话**：“文章越水，按键量的差距越小。主要是因为并击有很多的‘1键字词’。”
    
*   **键位频率分配的科学性**

    *   **解释**：如果按键随机排列，并击效率会很低。但通过按照**元素频率**分配键位，高频的元素组合在同一个键上的概率变大，使得单手平均按键量小于1.5（比如1.25），从而保证了高效率。
    
    *   **算账**：假如随机排列，左手1-2键+右手1-2键平均要3键，两击就是6键，比串击慢；但按频率排列后，平均按键量显著下降。

### 3. 重码率与思维连贯性（适合“想打”）

这是写作者最看重的一点，即不打断思路。

*   **限制条件拉满（加入声调）**

    *   **解释**：拼音本身重码率高，是因为声调信息丢失。并击（特别是拼读并击）把声调加上了，这相当于把拼音的限制条件“拉满了”，从而极大地降低了重码率。
    
    *   **数据对比**：双拼双形的选重率约为11%，而拼读并击可以降至2.2%。
    
    *   **原话**：“加上声调你会发现一个拼音的所有的东西全部加上了，限制条件拉满了，这样重码率就是最低的。”
    
*   **避免打断创作状态**

    *   **解释**：重码率低意味着写作者不需要停下来选词，也不容易打错，能够全身心沉浸在剧情或内容构思中（即“想打”）。
    
    *   **体验**：“我写小说的时候只想专注想剧情...打字词的时候要选重，会打断我的状态，所以我需要一个最低的重码率。”
    
*   **音节解析（跳过字形）**

    *   **解释**：音码打词时，大脑是对“音节”进行直接解析并转化为编码，中间跳过了具体“字”的环节，这比形码更适合边想边打。
    
    *   **原话**：“本质在于音码打词时是不完全需要知道是哪几个字组成词的，而是对音节进行解析。”
    
### 4. 适用速度区间与学习成本

*   **最适合的速度区间：160-180字/分**

    *   **解释**：并击的强项不在于追求极限的爆发速度（虽然空明码上限很高），而在于在最常用的160-180字/分区间内，能以最轻松的姿态保持输出。
    
    *   **原话**：“并击的优势并不是上限的输出，而是在适合的速度中保持有效的休息频率。”
    
*   **入门相对简单**

    *   **解释**：虽然空明码（形码）入门难，但像拼读这种纯音码并击方案，入门门槛相对较低。
    
    *   **原话**：“像拼读这种的方案 入门就简单很多。”
    
    *   **观点**：想打用音码更轻松，形码想打虽然练到极致也可以，但中间过程的练习成本和脑子痛苦程度远大于练音码。
    
*   **性价比总结**

    *   **解释**：综合来看，拼读并击利用“拼读拆分”提供了大部分性能（选重率4.5%，优于五笔的7.6%），如果做成26码元的串击方案还能用于手机，日常使用性价比极高。
    
    *   **原话**：“其他的带调拼音方案在性价比上不如用拼读的拆分，也不值得用。”
